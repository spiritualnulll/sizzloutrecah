Bot Configuration Documentation

This page explains each configuration variable in the bot's config.yaml file and provides warnings or notes where applicable.

DISCORD_TOKEN

DISCORD_TOKEN is the authentication key for your bot to connect with Discord's API.

Warning: Never share this token. If leaked, revoke it immediately from the Discord Developer Portal and generate a new one.

ALLOWED_SERVER_IDS

ALLOWED_SERVER_IDS is a list of server IDs where the bot is authorized to function. The bot will automatically leave any server not listed here.

Warning: Ensure this list is accurate to avoid unintended disconnections.

IMAGE_DIRECTORY

IMAGE_DIRECTORY specifies the directory where outreach-related images are stored. The default is image.

Ensure the directory exists and is writable to avoid errors.

DATA_FILE

DATA_FILE is the JSON file where outreach data is saved. The default file is outreach_data.json.

Make backups regularly to prevent data loss if you want.

Warning: This file gets big, so don't nuke it for that :P

EMBED_COLOR

EMBED_COLOR determines the default color of the embeds in bot messages. The default is #2f3136, which matches Discord's dark mode theme.

OWNER_IDS

OWNER_IDS is a list of user IDs that are granted elevated permissions for bot management, irrespective of their roles.

Warning: Only include trusted users here to prevent misuse.

INTERGRITY_SIGNOFF

INTERGRITY_SIGNOFF is a hash key used for verification and encryption tasks, ensuring secure operations within the bot.
